---
title: Подключаем библиотеку к странице
---

`Three.js` - это библиотека. Поэтому, для того чтобы ею воспользоваться, в первую очередь её необходимо подключить к `HTML` документу.

> Библиотека - код написанный сторонними разработчиками, предоставляющий некоторый интерфейс для взаимодействия с ним.

Давайте посмотрим как это можно сделать самым простым способом.

> Дальше в учебнике мы рассмотрим более продвинутые способы подключения, пока же сосредоточимся на самом главном, усложняя наши примеры только лишь по мере решения возникающих перед нами задач.

Для этого нам понадобится создать минимальный `HTML` документ.^[Максимально быстро позволяет сделать это аббревиатура (`abbreviation`) [`!`](https://docs.emmet.io/cheat-sheet/#:~:text=foo%3E%3C/foo%3E.-,!,-Alias%20of%20html) инструмента `emmet`, которая заменяется на шаблон с минимальным кодом `HTML` документа]

```html
<!doctype html>
<html>
  <head>
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <meta charset="utf-8" />
  </head>
  <body></body>
</html>
```

Самый простой способ - подключить библиотеку через `CDN`.

> По [ссылке](https://www.jsdelivr.com/package/npm/three) можно выбрать нужную версию и скопировать соответствующий `URL`. Так же можно воспользоваться любым другим более подходящим `CDN`.

Библиотека `Three.js` написана с использованием синтаксиса модулей. Поэтому, для того чтобы импортировать свойства и методы библиотеки в наш скрипт, нам нужно указать, что наш скрипт тоже является модулем.

```html
<!doctype html>
<html>
  <head>
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <meta charset="utf-8" />
    <script type="module"></script>
  </head>
  <body></body>
</html>
```

Далее, используя директиву `import` и `URL` ресурса в `CDN` импортируем всё содержимое в переменную.

```html
<!doctype html>
<html>
  <head>
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <meta charset="utf-8" />
    <script type="module">
      import * as THREE from "https://cdn.jsdelivr.net/npm/three@0.183.1/+esm";
    </script>
  </head>
  <body></body>
</html>
```

Чтобы убедиться, что библиотека подключилась корректно, после импорта в переменную `THREE`, её версия доступна через поле `THREE.REVISION` (так же в глобальном объекте `window` должно быть доступно поле `__THREE__` содержащее номер версии).

> В нашем случае - `183`

```html
<!doctype html>
<html>
  <head>
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <meta charset="utf-8" />
    <script type="module">
      import * as THREE from "https://cdn.jsdelivr.net/npm/three@0.183.1/+esm";
      console.log(THREE.REVISION);
    </script>
  </head>
  <body></body>
</html>
```

Библиотека подключена! Теперь мы можем взглянуть на то что есть у неё внутри!
Прежде чем двигаться дальше, посмотрим, есть ли что-нибудь, что можно было бы улучшить, в таком, самом простом коде.

# Улучшаем наш код

А вот и первое усложнение.
В дальнейшем, с ростом сложности разрабатываемого приложения, у нас возникнет искушение разбивать его на подмодули.
В каждом подмодуле, нам нужно будет импортировать библиотеку снова и снова.
При выходе новой версии, указание абсолютного пути до библиотеки в каждом подмодуле затруднит её обновление.
Для этого следуя принципу `LC&HC`^[Low Coupling & High Cohesion] разделим явное указание версии библиотеки от её использования.
Инструменты сборки, которые мы рассмотрим дальше, подумают об этом за нас. Но так как у нас их ещё нет, воспользуемся нативными возможностями.
Для этого объявим карту импортов.

```html
<!doctype html>
<html>
  <head>
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <meta charset="utf-8" />
    <script type="importmap">
      {
        "imports": {
          "three": "https://cdn.jsdelivr.net/npm/three@0.183.1/+esm"
        }
      }
    </script>
    <script type="module">
      import * as THREE from "three";
      console.log(THREE.REVISION);
    </script>
  </head>
  <body></body>
</html>
```

> `importmap` - это тип скрипта, содержащий `JSON` объект.
> Сейчас для нас важно свойство `imports`. Это объект, ключами которого являются спецификаторы модулей, именно они будут использоваться вместе с директивой `import`. Значениями же этого объекта является текст, который будет подставлен в директиву при её разрешении вместо спецификатора (это может быть либо относительный, либо абсолютный, как в нашем случае, путь до модуля)

Теперь вместо указания абсолютного пути, в каждом подмодуле, нам достаточно указать спецификатор. Обновление до обратно совместимой версии сопряжено с обновлением только лишь карты импортов.

# Итого

1. Проще всего поставить библиотеку через `CDN`.
2. Подключение через `importmap` делает код более модифицируемым.
3. Свойство `THREE.REVISION` (или `window.__THREE__`), подскажет что подключение прошло успешно.
